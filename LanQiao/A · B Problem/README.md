# A · B Problem

[题目链接](https://www.lanqiao.cn/problems/20535/learning/)

---

## 解题思路

我们把公式和代码先丢到一边，假设 $L=5$，我们用最通俗的“装箱子”来一步步拆解。

### 第一步：降维打击（把 4 个变量变成 2 个箱子）
题目要求：$X_A \cdot X_B + Y_A \cdot Y_B \le 5$

别看有 4 个字母，其实中间是加号，这完全就是两组独立的东西。
我们准备两个箱子：
* **左箱子**的体积 $M = X_A \cdot X_B$
* **右箱子**的体积 $N = Y_A \cdot Y_B$

现在题目变成了小学算术：**左箱子体积 + 右箱子体积 $\le 5$**。

---

### 第二步：一个箱子有几种拼法？（找约数）
假设左箱子的体积 $M = 4$。
因为 $M$ 是由 $X_A$ 和 $X_B$ 乘起来的，它能怎么拼？
* $1 \times 4$
* $2 \times 2$
* $4 \times 1$

一共 3 种拼法。你仔细看，这 3 种拼法，是不是刚好就等于数字 4 的**约数个数**（1、2、4）？
所以，我们用一个数组 `cnt` 来记录这个对应关系：`cnt[4] = 3`。
这就是为什么我们要算约数：**一个数的约数有几个，它对应的拼法就有几种。**

---

### 第三步：两个箱子怎么配对？（前缀和登场）
现在回看总目标：$M + N \le 5$。
因为 $X_A, X_B$ 都是正整数，乘积最小也是 1，所以两个箱子最小都是 1。

如果**左箱子 $M = 1$**：
右箱子 $N$ 只要不大于 4 就行（因为 $1 + N \le 5$）。
所以右箱子的体积可以是 1、2、3、4。
这时候总共有多少种拼法？
答案是：`左边拼法 × 右边拼法总和`
也就是：`cnt[1] * (cnt[1] + cnt[2] + cnt[3] + cnt[4])`

如果**左箱子 $M = 2$**：
右箱子 $N$ 只要不大于 3 就行（因为 $2 + N \le 5$）。
答案是：`cnt[2] * (cnt[1] + cnt[2] + cnt[3])`

你发现没有？括号里那一长串加法，每次都要重新算一遍，太浪费时间了。
所以我们提前算出一个“累计表”，这就是**前缀和 $S$**：
设 $S[4] = cnt[1] + cnt[2] + cnt[3] + cnt[4]$
那么上面的式子直接就变成了：
* $M=1$ 时，方案数 = `cnt[1] * S[4]`
* $M=2$ 时，方案数 = `cnt[2] * S[3]`
* $M=3$ 时，方案数 = `cnt[3] * S[2]`
* $M=4$ 时，方案数 = `cnt[4] * S[1]`

把它们全加起来，就是最终答案！你看，原本四个变量嵌套的死循环，直接被拍扁成了一层加法。

---

### 第四步：神仙操作（怎么快速求约数？）
按照正常思维，求 1 到 5000 每个数有几个约数，你要不停地去除，看能不能整除。这在计算机里非常慢。

算法大佬是怎么优化的？**别去找约数，去发红包（找倍数）。**

准备一个全是 0 的 `cnt` 数组。
* 数字 1 站出来说：“凡是我的倍数（1, 2, 3, 4, 5...），你们的约数里肯定都有我。我给你们每人发 1 个红包！” (`cnt` 全部 +1)
* 数字 2 站出来说：“凡是我的倍数（2, 4, 6, 8...），你们的约数里也有我。我给你们每人发 1 个红包！” (`cnt[2], cnt[4]...` 再次 +1)
* 数字 3 站出来说：“3, 6, 9... 过来领红包！”

这就是代码里那个双重循环：
```python
for i in range(1, L + 1):           # i 就是站出来发红包的数字
    for j in range(i, L + 1, i):    # j 就是 i 的倍数，步长为 i
        cnt[j] += 1                 # 领红包（约数个数 +1）
```
没有任何缓慢的除法和取余操作，纯加法，一路狂飙。
