# 1722. 执行交换操作后的最小汉明距离

[题目链接](https://leetcode.cn/problems/minimize-hamming-distance-after-swap-operations/)

---

### 【题解】并查集 + 哈希表计数：将动态交换转化为静态连通块

#### 一、 核心思路：交换的传递性与局部乱序

这道题最核心的突破口在于理解**交换操作的传递性**：

如果允许交换位置 $A$ 和 $B$，且允许交换位置 $B$ 和 $C$，那么经过多次操作，位置 $A$ 和位置 $C$ 的元素也能互相交换。

例如`allowedSwaps = [[0,4],[4,2],[1,3],[1,4]]`，则下标为`0, 1, 2, 3, 4`的元素可以最终互换为任意排列

这意味着，`allowedSwaps` 中提供的交换对，实际上把数组下标划分成了若干个**连通分量**。

在一个连通分量内部的任意几个下标，它们对应的元素可以通过无数次交换，实现**任意的重新排列**。

因此，原本复杂的“如何交换才能使距离最小”的动态问题，变成了一个静态匹配问题：

1. 我们不需要关心具体是怎么交换的。
2. 我们只需要知道哪些下标属于同一个“圈子”（连通分量）。
3. 在同一个圈子里，`source` 提供的元素集合，能有多少个与 `target` 需要的元素集合匹配上。匹配不上的，就是无法消除的汉明距离（`source` 和 `target` 有多少个元素是不同的）。

#### 二、 算法步骤解析

整体算法可以分为三个明确的阶段：

**1. 维护下标的连通性（并查集）**

- 初始化一个大小为 $N$ 的并查集，每个下标初始时各自为一个独立的集合。
- 遍历 `allowedSwaps`，将允许交换的两个下标通过 `union` 操作合并到同一个集合中。
- 这一步完成后，所有可以互相交换位置的下标，都会拥有相同的根节点（Root）。

**2. 按连通块提取下标集（分组）**

- 创建一个哈希表 `groups`，键为根节点，值为该根节点下辖的所有下标列表。
- 遍历 $0$ 到 $N-1$ 的所有下标，通过 `find(i)` 找到它们的根节点，并将其归类。
- **注意：** 从未出现在 `allowedSwaps` 中的下标，其根节点就是它自己，会被单独分入一个大小为 1 的分组中。这在逻辑上是完全自洽的，因为孤立的下标只能原地与 `target` 匹配，不能借助其他元素。

**3. 局部哈希计数与贪心匹配（计算汉明距离）**

- 遍历 `groups` 中的每一个连通分量（即一组下标）。
- 对于当前这一组下标，我们需要计算：在这些位置上，`source` 和 `target` 有多少个元素是不同的。
- **具体做法**：
  - 先遍历一次该组下标，用一个哈希表 `cnt` 记录 `source` 在这些位置上提供的所有元素的出现次数。
  - 初始设当前组的差异值 `diff` 为组的大小 `len(group)`。
  - 再遍历一次该组下标，查看 `target` 在这些位置上需要的元素。如果该元素在 `cnt` 中存在（即 `source` 的该连通块中恰好有这个数字），我们就把 `cnt[目标元素]` 减 1，同时把 `diff` 减 1（表示匹配成功了一对）。
- 将所有连通分量最终遗留的 `diff` 累加起来，即为最小汉明距离。

#### 三、 复杂度分析

- **时间复杂度：** $O(N \cdot \alpha(N) + M \cdot \alpha(N))$。

  其中 $N$ 为数组长度，$M$ 为 `allowedSwaps` 的长度，$\alpha$ 为阿克曼函数的反函数，可视为常数。

  - 构建并查集需要处理 $M$ 条边，耗时 $O(M \cdot \alpha(N))$。
  - 对所有下标进行分组，需要执行 $N$ 次 `find`，耗时 $O(N \cdot \alpha(N))$。
  - 最后遍历所有连通分量进行哈希计数，整体上相当于对 $N$ 个元素各自访问了两次（一次统计 `source`，一次匹配 `target`），耗时 $O(N)$。
  - 综合时间复杂度接近线性。

- **空间复杂度：** $O(N)$。

  - 并查集的 `parent` 数组需要 $O(N)$ 空间。
  - `groups` 字典存储了所有的分组下标，总计需要 $O(N)$ 空间。
  - `cnt` 哈希表在极端情况下（所有元素都在一个连通分量且各不相同）需要 $O(N)$ 空间。
  - 整体空间复杂度为 $O(N)$。
